# دليل المساهمة

شكراً لاهتمامك بالمساهمة في CryptoHub!

## البدء

1. قم بعمل **Fork** للمستودع على GitHub.
2. **استنسخ** نسختك محلياً وأنشئ فرع ميزة:

   ```bash
   git clone https://github.com/<you>/CryptoHub.git
   cd CryptoHub
   git checkout -b feature/my-change
   ```

3. أعدّ بيئة التطوير — انظر [SETUP.md](./SETUP.md).

## تسمية الفروع

| البادئة | الاستخدام |
|---------|-----------|
| `feature/` | ميزات جديدة |
| `fix/` | إصلاح الأخطاء |
| `docs/` | تغييرات التوثيق |
| `refactor/` | إعادة هيكلة الكود |
| `test/` | إضافة أو تحسين الاختبارات |

## رسائل الإيداع

اتبع مواصفات الإيداعات التقليدية:

```
feat: إضافة دعم مؤشر بولينجر باند
fix: إصلاح حساب نسبة شارب للبيانات الفارغة
docs: تحديث مرجع API
```

## معايير الكود

- **Python**: تنسيق/فحص Ruff، اختبارات `pytest`
- **Go**: `go fmt`، `go vet`، `go test`
- **TypeScript**: ESLint، `pnpm lint`، `pnpm build`

## الاختبار

- اكتب اختبارات لجميع الوظائف الجديدة.
- ضع اختبارات Python في `backend-python/tests/`.
- استخدم `pytest.mark.asyncio` للاختبارات غير المتزامنة.
- استهدف تغطية 80% على الأقل للكود الجديد.

## عملية طلب السحب

1. تأكد من نجاح CI.
2. املأ قالب PR بوصف واضح.
3. اطلب مراجعة من مشرف واحد على الأقل.
4. بعد الموافقة، يتم الدمج بطريقة squash-merge.

## التدويل

- التوثيق موجود بثماني لغات تحت `docs/{lang}/`.
- عند تحديث التوثيق الإنجليزي، يرجى الإشارة في PR.

## الإبلاغ عن المشكلات

استخدم GitHub Issues مع عنوان واضح وخطوات إعادة الإنتاج.

## الرخصة

بالمساهمة فإنك توافق على ترخيص مساهماتك بموجب رخصة MIT.

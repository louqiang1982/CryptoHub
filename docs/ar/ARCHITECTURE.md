# البنية المعمارية

يصف هذا المستند البنية المعمارية عالية المستوى لـ CryptoHub، بما في ذلك مسؤوليات الخدمات وتدفق البيانات والخيارات التقنية.

## نظرة عامة على النظام

يتبع CryptoHub **بنية الخدمات المصغرة** مع ثلاث خدمات رئيسية تتواصل عبر gRPC وREST، مدعومة بـ PostgreSQL للتخزين الدائم وRedis للتخزين المؤقت وتوزيع المهام.

## مسؤوليات الخدمات

### الواجهة الأمامية (Next.js 15 + React 19)

- عرض من جانب الخادم مع App Router
- تدويل بثماني لغات عبر `next-intl`
- تبديل السمات (فاتح / داكن / نظام) عبر `next-themes`
- رسوم بيانية احترافية مع KLineCharts وECharts
- تخطيط متجاوب (الهاتف، الجهاز اللوحي، سطح المكتب، الشاشات الكبيرة)

### خادم Go الخلفي

- بوابة API وإدارة المستخدمين ومصادقة JWT
- بث فوري عبر WebSocket
- عميل gRPC لاستدعاء خادم Python الخلفي

### خادم Python الخلفي (FastAPI)

- تحليل الذكاء الاصطناعي (LangChain + OpenAI)، حلقات الانعكاس، إجماع الأنظمة
- محرك الاستراتيجيات (تجميع، تحقق، تنفيذ في بيئة معزولة)
- محرك الاختبار الخلفي (شارب، سورتينو، الحد الأقصى للتراجع، عامل الربح)
- موفرو البيانات (عملات رقمية، أسهم، فوركس)

### عمال Celery

| العامل | الغرض |
|--------|-------|
| `pending_orders` | معالجة الأوامر المعلقة |
| `portfolio_monitor` | مراقبة المخاطر والتنبيهات |
| `market_data_collector` | جمع بيانات السوق دورياً |
| `reflection_worker` | انعكاس AI غير متزامن |
| `polymarket_worker` | تحديث بيانات أسواق التنبؤ |

## مخطط قاعدة البيانات

تُدار عبر ترحيلات Alembic. الجداول الرئيسية:

| الجدول | الوصف |
|--------|-------|
| `strategies` | استراتيجيات التداول المعرفة من المستخدم |
| `backtest_results` | نتائج الاختبار الخلفي والمقاييس |
| `indicators` | المؤشرات المخصصة ومؤشرات السوق |
| `ai_analyses` | سجلات تحليل الذكاء الاصطناعي |

# アーキテクチャ

CryptoHub の高レベルアーキテクチャ、サービスの責務、データフロー、技術選定について説明します。

## システム概要

CryptoHub は**マイクロサービスアーキテクチャ**を採用し、3 つの主要サービスが gRPC と REST で通信します。永続化には PostgreSQL、キャッシュとタスクブローカーには Redis を使用します。

```
        ┌────────────────┐  ┌───────────────┐  ┌────────────────┐
        │ フロントエンド    │  │ Go バックエンド  │  │Python バックエンド│
        │  (Next.js 15)  │  │  (Gin + WS)   │  │  (FastAPI)     │
        └───────┬────────┘  └──────┬────────┘  └──────┬─────────┘
                │                  │  gRPC            │
                │                  └────────┬─────────┘
                │                           │
           ┌────▼─────┐  ┌────────▼────┐  ┌─▼────────────┐
           │PostgreSQL │  │   Redis     │  │Celery Workers│
           └──────────┘  └─────────────┘  └──────────────┘
```

## サービスの責務

### フロントエンド（Next.js 15 + React 19）

- App Router によるサーバーサイドレンダリング
- `next-intl` で 8 言語の国際化
- `next-themes` でテーマ切替（ライト / ダーク / システム）
- KLineCharts・ECharts による専門チャート
- レスポンシブレイアウト（モバイル、タブレット、デスクトップ、大画面）

### Go バックエンド

- API ゲートウェイ、ユーザー管理、JWT 認証
- WebSocket リアルタイム配信
- gRPC クライアントで Python バックエンドを呼び出し

### Python バックエンド（FastAPI）

- AI 分析（LangChain + OpenAI）、リフレクションループ、アンサンブル
- 戦略エンジン（コンパイル、検証、サンドボックス実行）
- バックテストエンジン（シャープレシオ、ソルティノレシオ、最大ドローダウン、プロフィットファクター）
- データプロバイダー（暗号資産、株式、外国為替）

### Celery ワーカー

| ワーカー | 用途 |
|---------|------|
| `pending_orders` | 指値・逆指値注文の処理 |
| `portfolio_monitor` | リスク監視とアラート |
| `market_data_collector` | 定期的な市場データ取得 |
| `reflection_worker` | 非同期 AI リフレクション |
| `polymarket_worker` | 予測市場データの更新 |

## データベーススキーマ

Alembic マイグレーションで管理。主要テーブル：

| テーブル | 説明 |
|---------|------|
| `strategies` | ユーザー定義の取引戦略 |
| `backtest_results` | バックテスト結果と指標 |
| `indicators` | カスタム・マーケットプレイスインジケーター |
| `ai_analyses` | AI 分析レコード |

## セキュリティ

- JWT 認証（HS256）
- 戦略コードのサンドボックス化
- CORS ミドルウェア
- 環境変数ベースのシークレット管理

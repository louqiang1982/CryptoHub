# 아키텍처

CryptoHub의 상위 레벨 아키텍처, 서비스 책임, 데이터 흐름, 기술 선택에 대해 설명합니다.

## 시스템 개요

CryptoHub은 **마이크로서비스 아키텍처**를 채택하며, 세 가지 주요 서비스가 gRPC와 REST로 통신합니다. PostgreSQL로 영속성을, Redis로 캐싱과 태스크 브로커링을 담당합니다.

```
        ┌────────────────┐  ┌───────────────┐  ┌────────────────┐
        │   프론트엔드     │  │  Go 백엔드     │  │ Python 백엔드   │
        │  (Next.js 15)  │  │  (Gin + WS)   │  │  (FastAPI)     │
        └───────┬────────┘  └──────┬────────┘  └──────┬─────────┘
                │                  │  gRPC            │
                │                  └────────┬─────────┘
                │                           │
           ┌────▼─────┐  ┌────────▼────┐  ┌─▼────────────┐
           │PostgreSQL │  │   Redis     │  │Celery Workers│
           └──────────┘  └─────────────┘  └──────────────┘
```

## 서비스 책임

### 프론트엔드 (Next.js 15 + React 19)

- App Router를 사용한 서버 사이드 렌더링
- `next-intl`로 8개 언어 국제화
- `next-themes`로 테마 전환 (라이트 / 다크 / 시스템)
- KLineCharts와 ECharts를 이용한 전문 차트
- 반응형 레이아웃 (모바일, 태블릿, 데스크톱, 대형 화면)

### Go 백엔드

- API 게이트웨이, 사용자 관리, JWT 인증
- WebSocket 실시간 스트리밍
- gRPC 클라이언트로 Python 백엔드 호출

### Python 백엔드 (FastAPI)

- AI 분석 (LangChain + OpenAI), 리플렉션 루프, 앙상블
- 전략 엔진 (컴파일, 검증, 샌드박스 실행)
- 백테스트 엔진 (샤프 비율, 소르티노 비율, 최대 낙폭, 수익 팩터)
- 데이터 제공자 (암호화폐, 주식, 외환)

### Celery 워커

| 워커 | 용도 |
|------|------|
| `pending_orders` | 지정가/손절 주문 처리 |
| `portfolio_monitor` | 리스크 모니터링 및 알림 |
| `market_data_collector` | 주기적 시장 데이터 수집 |
| `reflection_worker` | 비동기 AI 리플렉션 |
| `polymarket_worker` | 예측 시장 데이터 갱신 |

## 데이터베이스 스키마

Alembic 마이그레이션으로 관리. 주요 테이블:

| 테이블 | 설명 |
|--------|------|
| `strategies` | 사용자 정의 거래 전략 |
| `backtest_results` | 백테스트 결과 및 지표 |
| `indicators` | 커스텀 및 마켓플레이스 인디케이터 |
| `ai_analyses` | AI 분석 기록 |

## 보안

- 환경 변수 기반 시크릿 관리

# Руководство по участию

Спасибо за интерес к участию в CryptoHub!

## Начало работы

1. Сделайте **Fork** репозитория на GitHub.
2. **Клонируйте** вашу копию и создайте ветку:

   ```bash
   git clone https://github.com/<you>/CryptoHub.git
   cd CryptoHub
   git checkout -b feature/my-change
   ```

3. Настройте среду разработки — см. [SETUP.md](./SETUP.md).

## Именование веток

| Префикс | Использование |
|---------|--------------|
| `feature/` | Новые возможности |
| `fix/` | Исправление ошибок |
| `docs/` | Изменения документации |
| `refactor/` | Рефакторинг кода |
| `test/` | Добавление и улучшение тестов |

## Сообщения коммитов

Следуйте спецификации Conventional Commits:

```
feat: добавить поддержку индикатора Bollinger Bands
fix: исправить расчёт коэффициента Шарпа для пустых данных
docs: обновить справочник API
```

## Стандарты кода

- **Python**: Ruff для форматирования/линтинга, `pytest` для тестов
- **Go**: `go fmt`, `go vet`, `go test`
- **TypeScript**: ESLint, `pnpm lint`, `pnpm build`

## Тестирование

- Пишите тесты для всех новых возможностей.
- Python-тесты размещайте в `backend-python/tests/`.
- Для асинхронных тестов используйте `pytest.mark.asyncio`.
- Стремитесь к покрытию не менее 80% нового кода.

## Процесс Pull Request

1. Убедитесь, что CI прошёл успешно.
2. Заполните шаблон PR с чётким описанием.
3. Запросите ревью минимум у одного мейнтейнера.
4. После утверждения — squash-merge.

## Интернационализация

- Документация ведётся на 8 языках в `docs/{lang}/`.
- При обновлении английской документации укажите это в PR.

## Отчёты об ошибках

Используйте GitHub Issues с чётким заголовком и шагами воспроизведения.

## Лицензия

Участвуя в проекте, вы соглашаетесь, что ваши вклады будут лицензированы под MIT License.

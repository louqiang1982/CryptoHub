# Архитектура

Описание высокоуровневой архитектуры CryptoHub: ответственности сервисов, потоки данных и выбор технологий.

## Обзор системы

CryptoHub использует **микросервисную архитектуру** с тремя основными сервисами, взаимодействующими через gRPC и REST. PostgreSQL обеспечивает хранение данных, Redis — кэширование и очередь задач.

## Ответственности сервисов

### Фронтенд (Next.js 15 + React 19)

- Серверный рендеринг через App Router
- Интернационализация на 8 языков через `next-intl`
- Переключение тем (светлая / тёмная / системная) через `next-themes`
- Профессиональные графики на KLineCharts и ECharts
- Адаптивная вёрстка (мобильные, планшеты, ПК, большие экраны)

### Go бэкенд

- API-шлюз, управление пользователями, JWT-аутентификация
- WebSocket для потоковой передачи в реальном времени
- gRPC-клиент для вызова Python бэкенда

### Python бэкенд (FastAPI)

- ИИ-анализ (LangChain + OpenAI), циклы рефлексии, ансамбль
- Движок стратегий (компиляция, валидация, песочница)
- Движок бэктестинга (коэффициенты Шарпа, Сортино, макс. просадка, фактор прибыли)
- Провайдеры данных (криптовалюты, акции, форекс)

### Celery воркеры

| Воркер | Назначение |
|--------|-----------|
| `pending_orders` | Обработка отложенных ордеров |
| `portfolio_monitor` | Мониторинг рисков и оповещения |
| `market_data_collector` | Периодический сбор рыночных данных |
| `reflection_worker` | Асинхронная рефлексия ИИ |
| `polymarket_worker` | Обновление данных рынков прогнозов |

## Схема базы данных

Управляется миграциями Alembic. Основные таблицы:

| Таблица | Описание |
|---------|----------|
| `strategies` | Пользовательские торговые стратегии |
| `backtest_results` | Результаты бэктестов и метрики |
| `indicators` | Пользовательские и рыночные индикаторы |
| `ai_analyses` | Записи ИИ-анализов |

## Безопасность

- JWT-аутентификация (HS256)
- Песочница для кода стратегий
- Промежуточное ПО CORS
- Управление секретами через переменные окружения

# 架构说明

本文档描述 CryptoHub 的高层架构，包括服务职责、数据流和技术选型。

## 系统概览

CryptoHub 采用**微服务架构**，三个主要服务通过 gRPC 和 REST 通信，后端由 PostgreSQL 提供持久化存储，Redis 负责缓存和任务调度。

```
                        ┌────────────────────────────────┐
                        │          负载均衡器              │
                        └───────────┬────────────────────┘
                                    │
                 ┌──────────────────┼──────────────────┐
                 │                  │                   │
        ┌────────▼───────┐ ┌───────▼────────┐ ┌────────▼───────┐
        │     前端        │ │   Go 后端       │ │  Python 后端    │
        │  (Next.js 15)  │ │  (Gin + WS)    │ │  (FastAPI)     │
        └────────────────┘ └───────┬────────┘ └────────┬───────┘
                                   │  gRPC             │
                                   └──────────┬────────┘
                                              │
                            ┌─────────────────┼─────────────────┐
                            │                 │                 │
                     ┌──────▼──────┐  ┌───────▼──────┐ ┌───────▼──────┐
                     │ PostgreSQL  │  │    Redis     │ │ Celery 工人   │
                     └─────────────┘  └──────────────┘ └──────────────┘
```

## 服务职责

### 前端（Next.js 15 + React 19）

- 使用 App Router 进行服务端渲染
- 通过 `next-intl` 实现国际化（8 种语言）
- 通过 `next-themes` 切换主题（亮色 / 暗色 / 跟随系统）
- 使用 KLineCharts 和 ECharts 提供专业图表
- 响应式布局，支持手机、平板、桌面和大屏

### Go 后端

- **API 网关** — 用户管理、认证和通用 CRUD 的 REST 端点
- **WebSocket 服务** — 实时价格推送和订单更新
- **gRPC 客户端** — 调用 Python 后端进行 AI 分析和策略执行

### Python 后端（FastAPI）

- **AI 分析** — 基于 LangChain 和 OpenAI 的 LLM 驱动分析，支持反思循环和集成共识
- **策略引擎** — 在沙箱环境中编译、验证和运行用户编写的 Python 策略
- **回测引擎** — 在历史数据上模拟策略，提供完整指标（夏普、索提诺、最大回撤、盈利因子）
- **数据提供者** — 从加密交易所（ccxt）、股票市场（Yahoo Finance）和外汇获取数据
- **gRPC 服务** — 向 Go 后端暴露服务接口

### Celery 工人

由 Celery 管理的后台任务，使用 Redis 作为消息中间件：

| 工人 | 用途 |
|------|------|
| `pending_orders` | 处理挂单（限价单/止损单） |
| `portfolio_monitor` | 检查风险限制并生成告警 |
| `market_data_collector` | 定期获取和缓存市场数据 |
| `reflection_worker` | 异步运行 AI 反思循环 |
| `polymarket_worker` | 刷新预测市场数据 |

## 数据库架构

通过 Alembic 迁移管理。主要表：

| 表 | 说明 |
|---|------|
| `strategies` | 用户定义的交易策略 |
| `backtest_results` | 回测结果及指标 |
| `indicators` | 自定义和市场指标 |
| `ai_analyses` | AI 分析记录 |

## 安全性

- Go 后端管理的 JWT 认证（HS256）
- 策略代码沙箱化（禁止危险导入）
- 两个后端均配置 CORS 中间件
- 基于环境变量的密钥管理
